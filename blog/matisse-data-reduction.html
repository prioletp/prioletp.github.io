Data Reduction with the MATISSE Pipeline

📡

VLTI/MATISSE is a four-telescope beam combiner operating in the L, M and N bands (3–13 µm). It provides spectrally dispersed visibilities, closure phases, and differential phases, making it a powerful tool for studying circumstellar environments. This tutorial walks you through the essential steps of reducing MATISSE data using the ESO pipeline.

Prerequisites

Before starting, make sure you have the following installed:

• The ESO Reflex environment or the command-line EsoRex tool.
• The MATISSE pipeline recipes (available from the ESO pipeline page).
• Your raw data downloaded from the ESO archive, including both science and calibrator observations.
• The associated static calibration files.

Step 1: Organising Your Data

MATISSE raw data consists of FITS files organised by observing block (OB). Each OB includes target acquisitions, fringe tracking files, photometric calibrations, and interferometric exposures. Start by sorting your data into directories by target and OB. The ESO archive provides an association tree that links raw science files to their corresponding calibrations.

Step 2: Running the Pipeline Recipes

The MATISSE pipeline is composed of several recipes that must be run in sequence:

1. mat_raw_estimates: Computes raw coherent flux and photometric estimates from the interferometric exposures.
2. mat_cal_oifits: Calibrates the raw observables using a calibrator star with known diameter, producing calibrated OIFITS files with visibilities, closure phases, and differential phases.

In practice, you will also need to run the dark and flat-field recipes (mat_raw_dark, mat_raw_flat) to produce master calibration frames.

Step 3: Quality Control

After reduction, carefully inspect your calibrated data. Key things to check:

• Transfer function stability: Plot the calibrator visibilities over the night. Large variations indicate poor atmospheric conditions or instrumental issues.
• Closure phase consistency: For unresolved calibrators, closure phases should be close to zero.
• Spectral coverage: Check that all spectral channels have been reduced properly, especially at the edges of the atmospheric windows.

Understanding Correlations

One important aspect when working with MATISSE data — especially for faint circumstellar emission — is that data points are not independent. Spectral channels within the same baseline tend to be strongly correlated, and correlations also exist between baselines sharing a telescope. In my work (Priolet et al., submitted), I developed an empirical approach to estimate correlation matrices for MATISSE data. Accounting for these correlations is crucial for accurate model fitting.

Tips & Common Pitfalls

• Always reduce your data with the same pipeline version for consistency.
• Be careful with the chopping/nodding subtraction in N-band — incorrect background subtraction can introduce systematic biases.
• When working with faint sources, consider using the coherent integration mode to boost SNR.
• The OIFITS 2 standard supports correlation matrices — consider using this format for your reduced data.
