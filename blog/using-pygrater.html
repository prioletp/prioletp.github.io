Radiative Transfer Modelling with pyGRaTeR

💻

pyGRaTeR is a Python radiative transfer code I am developing, specifically designed for modelling optically thin debris disks and their inner regions (exozodiacal dust). It is based on the IDL code GRaTeR (Augereau et al. 1999), but rewritten in Python with extended features and an open-source approach. This tutorial introduces the core concepts and shows you how to get started.

What pyGRaTeR Does

pyGRaTeR computes the thermal emission and scattered light from dust grains in optically thin environments. The key capabilities include:

• Mie theory: Computing the absorption and scattering efficiencies of spherical dust grains as a function of grain size, composition, and wavelength.
• Thermal emission: Calculating the equilibrium temperature of grains at a given distance from the star, and their resulting spectral energy distribution (SED).
• Scattering: Modelling the scattered light contribution including the scattering phase function.
• N-body inputs: Incorporating size and spatial distributions from dynamical N-body simulations (e.g., from the tools developed by Sezestre and Péronne at IPAG).

Getting Started

pyGRaTeR requires Python 3.8+ and depends on standard scientific packages:

• numpy and scipy for numerical computations
• matplotlib for plotting
• astropy for FITS I/O and unit handling

Core Workflow

A typical pyGRaTeR workflow involves three main steps:

1. Define the stellar properties: Specify the star's luminosity, temperature, and distance. pyGRaTeR uses these to compute the radiation field incident on the dust grains.

2. Set up the dust model: Choose the grain composition (e.g., astronomical silicates, carbonaceous grains), the size distribution (typically a power law dn/da ∝ a−q), and the spatial distribution (radial density profile). You can also import distributions from N-body simulations.

3. Compute observables: pyGRaTeR computes the grain temperatures, SED, and surface brightness profiles. These can then be compared directly to observations — SEDs from photometry, or visibilities from interferometry.

Modelling Exozodiacal Dust

For exozodiacal dust, the emission is very faint compared to the star (typically ~1% of the stellar flux). This means:

• The dust is in the optically thin regime — each grain radiates independently, and self-absorption is negligible. This makes the radiative transfer much simpler and faster.
• The key observable from interferometry is the visibility drop — the circumstellar emission reduces the fringe contrast by a few percent compared to a point source.
• pyGRaTeR can compute model visibilities at any baseline and wavelength, allowing direct comparison with VLTI data.

Comparison with Other Codes

Unlike general-purpose radiative transfer codes (e.g., RADMC-3D, MCFOST), pyGRaTeR is optimised for optically thin sources. This makes it:

• Much faster — no need for Monte Carlo photon packets or iterative temperature calculations.
• Tailored to the specific observables of interest (interferometric visibilities, SEDs).
• Easy to couple with dynamical models for the dust production and transport.

The trade-off is that it cannot handle optically thick environments (e.g., protoplanetary disks), where multiple scattering and absorption matter.

What's Coming

pyGRaTeR is under active development. Upcoming features include:

• Full integration of N-body dynamical simulation outputs
• Optimised grain property calculations for large parameter grids
• Public release on GitHub with documentation and example notebooks
